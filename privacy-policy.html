DropToPC

Privacy Policy

Effective date: 2026-03-26

This draft is based on the current code and packaging in this repository.

1. What DropToPC Does

DropToPC is a Windows desktop app that receives files from a phone browser through a locally hosted receiver on the PC. In packaged builds, the receiver uses HTTPS with a locally generated self-signed certificate.

2. Information the App Handles

DropToPC can process:

  • Files you choose to upload from your phone.
  • File metadata such as file name, MIME type, and size.
  • Transfer metadata such as start time, completion time, status, and source LAN IP address.
  • Session data such as the one-time session token, whether a PIN is required, and session expiration time.
  • App settings such as the receive folder, size limits, security settings, and optional privacy/support links.
  • Security metadata such as Windows Defender scan status, scan messages, and quarantine status.
  • User-triggered support-bundle data such as diagnostics, recent local logs, sanitized settings, and sanitized transfer history.

3. Network Behavior

  • The phone upload page and upload API are hosted by the PC.
  • In packaged builds, transfer traffic between the phone browser and the PC receiver uses HTTPS.
  • The app source inspected for this release does not contain runtime analytics, crash reporting, advertising SDKs, automatic update checks, CDN calls, or developer-managed cloud relay services.
  • The mobile upload page is controlled by a one-time session URL token.
  • Any device that can reach the PC and knows the full session URL can load the upload page.
  • Successful uploads can also require the current session PIN and, by default, a same-subnet network check.

4. Local Storage

DropToPC stores data locally on the PC, including:

  • Accepted files in the selected receive folder.
  • Temporary in-progress upload files in <receive-folder>/_incoming before they are moved into the final destination.
  • Threat-detected files in <receive-folder>/_quarantine when quarantine is enabled.
  • Settings in settings.json under Electron userData.
  • Transfer history in history.json under Electron userData.
  • Local receiver certificate material in tls/receiver-cert.json under Electron userData.
  • Local operational logs in logs/droptopc.log and logs/droptopc.previous.log under Electron userData.
  • Optional support bundle export files that the user chooses to save.

5. Security Controls

DropToPC currently includes:

  • HTTPS transport in packaged builds.
  • A one-time session URL token.
  • An optional 6-digit PIN, enabled by default.
  • Session expiration, 10 minutes by default.
  • Optional same-subnet enforcement, enabled by default.
  • Upload rate limits and concurrent-upload limits.
  • Filename sanitization and duplicate-safe final naming.
  • Optional Windows Defender scanning.
  • Optional quarantine for detected threats.
  • Mark-of-the-Web tagging on uploaded, quarantined, and restored files.

DropToPC does not automatically install the local receiver certificate into the phone or PC trust store. A phone browser may show a certificate warning the first time it connects to the receiver.

6. User Controls

Users can:

  • Choose the receive folder.
  • Start or stop the receiver.
  • Regenerate the upload session.
  • Clear transfer history.
  • Export a support bundle.
  • Restore or delete quarantined files.
